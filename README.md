# Сборщик протокола

Desktop-приложение (Windows / macOS) для сборки общего протокола из отдельных выписок (DOCX / PDF / изображения).

## Что делает

1. Принимает 50+ файлов-выписок из протокола.
2. Из каждого файла извлекает блок `1.X.` (заголовок → таблица → Предложено → голосование → Принято решение → Единогласно).
3. Перенумеровывает пункты: 1-й файл → `1.1.`, 50-й → `1.50.`.
4. Собирает в один `Протокол_сформированный.docx` с общей шапкой.

## macOS — быстрый старт

### 1. Требования

- **Python 3.11+** (системный Python 3.14 на macOS не поддерживается из-за бага с libexpat):

```bash
brew install python@3.11
```

- **Tesseract** (только для PDF-сканов и изображений):

```bash
brew install tesseract tesseract-lang
```

### 2. Первоначальная настройка

```bash
cd protocol-builder
./setup_mac.sh
```

Создаёт `venv/` и устанавливает все зависимости.

### 3. Запуск

```bash
./run_mac.sh
```

### 4. Тесты

```bash
./run_tests.sh
```

### 5. Сборка .app (только на macOS)

```bash
./build_mac.sh
# Результат: dist/ПротоколСборщик.app
open dist/ПротоколСборщик.app
```

## Рабочий процесс

1. **Вкладка «Файлы»** — добавить выписки (кнопка / папка / drag & drop).
   - Зелёный = Успешно (confidence ≥ 85%)
   - Жёлтый = Нужна проверка (50–84%)
   - Красный = Фрагмент не найден / Ошибка
   - Кнопки ↑↓ меняют порядок пунктов в итоговом документе.
2. **Вкладка «Шапка протокола»** — заполнить реквизиты; кнопка «Заполнить из первого DOCX» пробует автозаполнение.
3. **Вкладка «Предпросмотр»** — проверить извлечённые фрагменты, при необходимости исключить.
4. **Вкладка «Сборка»** — «Сформировать протокол» → выбрать путь → «Открыть файл».

## Статусы и оценка уверенности

| Маркер                               | Вес |
| ------------------------------------ | --- |
| Заголовок `1.X.`                     | +20 |
| Таблица                              | +20 |
| «Предложено»                         | +15 |
| «В результате голосования»           | +15 |
| «Принято решение»                    | +15 |
| Конечный якорь («Единогласно» и др.) | +15 |

- **≥ 85** → Успешно — включается в протокол автоматически
- **50–84** → Нужна проверка — включается, но помечается в предпросмотре
- **< 50** → Не найден — выносится в раздел «Файлы, требующие ручной проверки»
- **Ошибка** — файл повреждён или не открывается; выносится в тот же раздел

## Форматы входных файлов

| Формат               | Метод                                        | Таблицы                 |
| -------------------- | -------------------------------------------- | ----------------------- |
| DOCX                 | XML-парсинг (lxml), сохраняет форматирование | Полностью с границами   |
| PDF (текстовый слой) | pdfplumber / PyMuPDF                         | Текстовое представление |
| PDF (скан)           | Tesseract OCR                                | Текстовое представление |
| JPG / PNG / BMP      | Tesseract OCR                                | Текстовое представление |
